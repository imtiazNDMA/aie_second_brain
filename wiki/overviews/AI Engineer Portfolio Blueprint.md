# AI Engineer Portfolio Blueprint

What to publish so hiring managers can verify capability quickly.

## Required Artifacts Per Flagship Project

- Problem framing and system diagram
- Reproducible implementation (clean repo, setup steps)
- Evaluation metrics and baseline comparisons
- Tradeoff analysis (latency, quality, cost)
- Failure analysis and next-iteration plan

## Publishing Standard

- One concise technical writeup per project
- One short demo clip or walkthrough
- One "what I learned" note linked to [[AI Engineer Weekly Review]]

## Portfolio Readiness Checklist

- 4 complete flagship projects
- 2 projects with explicit eval harnesses
- 2 projects with deployment/operations evidence
- At least 8 public technical artifacts
