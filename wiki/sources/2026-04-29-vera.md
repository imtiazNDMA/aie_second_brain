# VERA: Validation and Enhancement for Retrieval Augmented Systems

## Summary
System that evaluates and enhances both retrieved context and LLM-generated responses using an evaluator-cum-enhancer LLM. Improves response adherence and relevance for both small and large models.

## Key Takeaways

### VERA Architecture
Three-stage process:
1. **Retrieval Requirement Check**: Determines if external context is needed
2. **Context Evaluation/Correction**: Edits retrieved context to eliminate redundancy
3. **Response Evaluation/Correction**: Splits response into atomic statements, checks grounding

### Core Metrics

#### Response Adherence
Measures extent to which LLM's response is grounded in provided context:
$$A_{\text{response}} = \frac{1}{n} \sum_{i=1}^{n} g(s_i)$$
where $g(s_i) = 1$ if grounded, $0$ otherwise.

#### Response Relevance
Amount of response information relevant to answering the query.

#### Context Relevance
Amount of retrieved context pertinent to answering the query:
$$R_{\text{retrieval}} = \frac{|C'|}{|C|}$$
where $C'$ is edited context, $C$ is original.

### VERA Algorithms

**Context Editing** (Algorithm 2):
- Eliminate redundant information
- Score: $r(s_i) = 1$ if relevant, $0$ if not
- Return edited context $C'$

**Response Editing** (Algorithm 4):
- Split into atomic statements
- Classify: derivable, inferable, inaccurate
- Remove ungrounded statements
- Return final response $R'$

### Experimental Results

| Model | SQuAD 2.0 Acc | DROP Acc |
|-------|-------------------|----------|
| Mistral-7B | 0.416 | 0.432 |
| Mistral-7B + VERA | **0.582** (+20%) | **0.752** (+15%) |
| GPT-3.5-turbo | 0.490 | 0.696 |
| GPT-3.5-turbo + VERA | **0.640** (+15%) | **0.764** (+10%) |
| GPT-4o | 0.582 | 0.816 |
| GPT-4o + VERA | **0.690** (+5%) | **0.854** (+10%) |

### Downstream Tasks
- **World War II Wikipedia**: Tests adherence to provided context
- **Apple 10-K Report**: Financial data processing
- **SQuAD 2.0**: Reading comprehension
- **DROP**: Discrete reasoning over paragraphs

### Advantages
- **Model-agnostic**: Works with any LLM
- **Few-shot prompting**: No retraining required
- **Comprehensive**: Evaluates both retrieval and generation
- **Effective**: Significant improvements across model sizes
