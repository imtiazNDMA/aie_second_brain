# Self-RAG: Learning to Retrieve, Generate, and Critique Through Self-Reflection

## Summary
Introduces Self-Reflective Retrieval-Augmented Generation (Self-RAG) that enhances LLM quality and factuality through on-demand retrieval and self-reflection using special reflection tokens.

## Key Takeaways

### Core Innovation
Self-RAG trains a single LM to:
1. **Adaptively retrieve** passages on-demand using Retrieve tokens
2. **Generate** task outputs with retrieved context
3. **Self-critique** outputs using reflection tokens

### Reflection Tokens
Special tokens inserted into model vocabulary:
- **Retrieve**: {yes, no, continue} — decidess when to retrieve
- **ISREL**: {relevant, irrelevant} — evaluates passage relevance
- **ISSUP**: {fully supported, partially, no support} — checks if output is supported
- **ISUSE**: {5, 4, 3, 2, 1} — rates overall utility

### Training Process
1. **Critic model**: GPT-4 generates reflection tokens for training data
2. **Distillation**: Train smaller model to predict tokens
3. **End-to-end**: Generator learns to produce text + reflection tokens

### Inference Algorithm
- **Adaptive retrieval**: Predict Retrieve token at each step
- **Parallel processing**: Process multiple passages simultaneously
- **Segment-level beam search**: Select best continuation using weighted sum of reflection token probabilities
- **Customizable**: Adjust weights for different criteria at test time

### Experimental Results
Comparing 7B and 13B Self-RAG models:
- **Outperforms** ChatGPT and Llama2-chat on Open-domain QA
- **Superior factuality** on reasoning and fact verification
- **Better citation accuracy** for long-form generations
- **Beats** retrieval-augmented Llama2-chat and Alpaca

### Advantages Over Traditional RAG
- **Adaptive retrieval**: Only retrieves when necessary
- **Self-evaluation**: Critiques own outputs
- **Controllability**: Customize behavior at inference without retraining
- **Citation accuracy**: Provides attributions for each segment
