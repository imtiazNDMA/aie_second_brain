# Grokking Machine Learning

## Summary
Intuitive, math-light introduction to machine learning covering core algorithms with visual explanations and real-world examples. By popular YouTube educator Luis Serrano.

## Key Takeaways

### The Remember-Formulate-Predict Framework
The core mental model for ML:
1. **Remember**: Collect past examples (training data)
2. **Formulate**: Extract patterns, create rules (model)
3. **Predict**: Apply rules to new situations

### Supervised Learning
- **Linear Regression**: Fit line to predict continuous values (see [[Linear Regression]])
- **Logistic Regression**: Sigmoid for binary classification (see [[Logistic Regression]])
- **Decision Trees**: Split data based on questions (see [[Decision Trees]])
- **Naive Bayes**: Probabilistic classification (see [[Naive Bayes]])
- **Support Vector Machines**: Find optimal hyperplane (see [[Support Vector Machines]])

### Unsupervised Learning
- **Clustering**: Group similar data points (K-Means)
- **Dimensionality reduction**: PCA for feature compression
- **Association rules**: Find patterns (market basket analysis)

### Neural Networks
- **Perceptron**: Single neuron with weight update
- **Multi-layer networks**: Hidden layers for complex patterns
- **Keras implementation**: High-level neural network API
- **Applications**: Regression and classification with NNs

### Ensemble Methods
- **Bagging**: Bootstrap aggregating (Random Forest)
- **Boosting**: AdaBoost, Gradient Boosting
- **XGBoost**: Extreme gradient boosting
- **Benefits**: Reduced overfitting, improved accuracy

### Model Evaluation
- **Train/test split**: Avoid overfitting
- **Cross-validation**: K-fold validation
- **Metrics**: Accuracy, precision, recall, F1-score
- **ROC curves**: Visualize classifier performance

## Practical Examples
- **Housing prices**: Linear regression with feature engineering
- **Spam detection**: Naive Bayes with text features
- **App recommendations**: Decision trees with user data
- **Titanic survival**: End-to-end ML pipeline
