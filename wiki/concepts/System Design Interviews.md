# System Design Interviews

## Definition

Open-ended architecture interviews where candidates design scalable systems under ambiguous requirements, explain tradeoffs, and communicate decisions clearly.

## Core Workflow

- Clarify goals, users, and constraints.
- Define scope and non-goals.
- Estimate scale (traffic, storage, read/write mix, latency targets).
- Draw a high-level architecture and identify bottlenecks.
- Dive into critical components and justify tradeoffs.
- Plan a staged path from [[Day 0 Architecture]] to [[Day N Architecture]].
