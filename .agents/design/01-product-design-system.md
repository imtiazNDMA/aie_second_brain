# Product Design System Guidelines

## Goals

- Make knowledge workflows feel intentional and trustworthy.
- Prioritize readability, hierarchy, and focus.
- Keep interaction patterns consistent across all surfaces.

## Foundations

- Define design tokens for color, spacing, radius, shadow, and motion.
- Define typography scale for page, search, and chat contexts.
- Standardize icon usage and semantic color meanings.

## Components to Standardize Early

- App shell and navigation
- Search input and result item
- Citation badge and evidence card
- Markdown renderer styles
- Empty/error/loading states
- Toast and inline feedback

## Motion Principles

- Use subtle transitions to preserve orientation.
- Avoid decorative animation that slows task completion.
- Animate relationship changes in graph with purpose.

## Content Design

- Prefer concise labels and explicit actions.
- Explain model uncertainty in plain language.
- Keep citation text short but discoverable.
